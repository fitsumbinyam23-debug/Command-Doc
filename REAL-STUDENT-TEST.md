# Real Student Test Guide

Test as a student, without opening developer tools or reading source code.

## Beginner pass

1. Open the public app in a private browser session.
2. From Home, choose **Learn Commands** and select one vendor track.
3. Open the first available lesson. Decide whether the purpose, syntax, healthy output, faulty output, and next step are understandable before revealing guidance.
4. Continue to Switch Lab and type the requested command yourself. Do not use the Run button unless testing accessibility.
5. Record whether the next action is clear, feedback is helpful, and progress feels earned.

## Experienced technician pass

1. Choose a specific vendor track and confirm the displayed syntax, prompt, interface format, and output match that vendor's content.
2. In Diagnose, use an exact command, an abbreviation, incomplete input, and an unrecognized input.
3. In Switch Lab, inspect state first, make one supported change, verify it, then save or roll it back.
4. Test a returning session by refreshing and checking that the simulated state and progress are restored.

## Evaluation notes

For each screen, record one status: `Working`, `Partially working`, `Not working`, `Not implemented`, or `Blocked`.

Judge clarity of navigation, command explanations, output interpretation, errors, hints, vendor separation, route-to-lesson links, terminal focus, and whether the result feels safe and professional. Do not infer a feature is working from a button label: complete the visible workflow.
